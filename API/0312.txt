커널 오브젝트
- 제어를 위해서
 특정 커널의 핸들을 가지고 있어야 하고, 권한이 있어야 한다.

- 핸들 값
 데이터 형이 모두 HANDLE이다.

- 객체 해제 과정
 closwHandle로 사라지는데, 보통 자기 스스로 파괴되고 그 판단 기준은 자신을 사용하는 객체가 없다면 스스로 파괴한다.

- 프로그램의 메모리 사용 구역
 32bit에서는 최대 4gb의 크기를 이용 할 수있다.
 2gb는 os가 쓰고
 2gb는 user가 쓸 수 잇다.
 이 안에 스택, 큐 , 전역공간, 코드 메모리 등이 담긴다.

- 페이징 파일
 파일을 메모리 처럼 사용하는 것 = 가상메모리 
 
단 현재 실행하고 있는 파일은 반드시 메모리 상에 존재 해야한다.

- 프로세스 란
 작업 단위 이며 커널 객체의 가상 핸들? 아무튼 주소다. 또 pId로 프로세스 아이디를 갖게 된다
 메모리 상에 존재하는 프로그램이다.


- 스레드 란
 프로세스의 코드를 읽고 수행하는 행동이다. 코드를 한줄 한줄 이행하는 녀석
 즉 cpu의 스레드를 의미하는 것
 스레드는 하나의 프로세스에 하나의 스레드가 존재 할때 가장 완벽한 상태다

- 커널 오브젝트가 사라질때
 커널 오브젝트는 카운트가 0이 될때 signal로 자기 상태를 변경함. 
 이후 os가 이 커널 오브젝트를 삭제함


맵핑 필드는 어디에 존재하는가? os 2Gb에 존재한다.
