tcp 네트워크 과정

서버
1.소켓 생성 socket
2.소켓에 주소 할당 bind
- ip랑 port가 할당

3.네트워크와 소켓을 연결 listen

4.네트워크의 데이터를 받기 위해 대기하는 상태 accept
- 함수가 멈춰 있는 상황 


5.accept가 성공하면 pc 내부에 또다른 socket이 생성되고 이 socket은 상대방을 나타내는 소켓이다. = 통신용 소켓


6.네트워크를 중단하겠다 - cloasesocket 
 - 열어놓은 모든 소켓을 닫기 위해서 기존에 만든 소켓도 닫고, 통신용 소켓도 닫는다.

클라이언트
1, 소켓 생성 socket
2. 소캣에 주소 할당 connect때 한다능 그리고 이건 통신 소켓이라능
 - 그래서 자동적으로 os가 ip랑 port를 할당한다능
